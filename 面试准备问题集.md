### Q2.2：C++中指针和引用的区别是什么？

**回答思路：** 从定义、语法、使用场景对比

**参考回答：**

"**指针和引用的区别：**

| 特性 | 指针 | 引用 |
|------|------|------|
| 定义 | 存储地址的变量 | 变量的别名 |
| 是否为空 | 可以是nullptr | 不能为空（必须初始化） |
| 是否可以重新赋值 | 可以指向不同对象 | 不能改变引用的对象 |
| 是否需要解引用 | 需要`*`解引用 | 直接使用 |
| 是否有独立地址 | 有自己的地址 | 和引用对象共享地址 |

**项目中的应用：**
```cpp
// 指针：在CKernel中使用指针管理UI对象
Dialog* m_dialog;  // 指针，可以延迟初始化

// 引用：在函数参数中使用引用避免拷贝
void setHostInfo(int id, QString& name);  // 引用，避免拷贝
```

**选择原则：**
- 如果需要**重新赋值**或**可能为空**，使用指针
- 如果**不会变化**且**一定有效**，使用引用"

### Q2.5：项目中使用了哪些STL容器？为什么选择它们？

**回答思路：** 列举使用的容器 + 说明选择理由

**参考回答：**

"**项目中使用的STL容器：**

```cpp
std::stack<QPoint> m_undoStack;  // 悔棋栈（如果有悔棋功能）

**选择理由：**
- 悔棋功能需要LIFO（后进先出）特性

## 三、Qt框架相关问题

### Q3.1：Qt的信号-槽机制是什么？是如何实现的？

**回答思路：** 原理说明 + 项目中的应用 + 不同连接方式的区别

**参考回答：**

"**信号-槽机制是Qt的核心特性，用于对象间的通信。**

**工作原理：**
1. **moc（元对象编译器）**：预处理带有`Q_OBJECT`宏的类，生成元对象代码
2. **信号触发**：当信号被发射时，Qt的元对象系统会调用所有连接的槽函数
3. **连接类型**：
   - `Qt::AutoConnection`：自动选择（默认）
   - `Qt::DirectConnection`：同步调用（同一线程）
   - `Qt::QueuedConnection`：异步调用（跨线程，通过事件循环）

**项目中的应用：**
```cpp
// 在roomdialog.cpp中
connect(ui->widget, SIGNAL(SIG_PIECEDOWN(int,int,int)), 
        this, SIGNAL(SIG_PIECEDOWN(int,int,int)));
```
这是将棋盘控件的信号转发给CKernel处理。

**新语法（Qt5）：**
```cpp
// 编译时检查，更安全
connect(ui->widget, &FiveInLine::SIG_PIECEDOWN,
        this, &roomDialog::SIG_PIECEDOWN);
```

**底层实现（简化）：**
- Qt使用`moc`生成信号-槽的元数据
- 信号函数由`moc`自动生成，内部调用`QMetaObject::activate()`
- `activate()`遍历所有连接的槽函数并调用"

### Q3.3：Qt中多线程如何实现？项目中需要多线程吗？

**回答思路：** 说明Qt的多线程方式 + 分析项目需求

**参考回答：**

"**Qt中实现多线程的方式：**

**1. QThread + moveToThread（推荐）**
```cpp
class Worker : public QObject
{
    Q_OBJECT
public slots:
    void doWork() { /* 耗时操作 */ }
};

QThread* thread = new QThread;
Worker* worker = new Worker;
worker->moveToThread(thread);
connect(thread, &QThread::started, worker, &Worker::doWork);
thread->start();
```

**2. 继承QThread（旧方式，不推荐）**
```cpp
class MyThread : public QThread
{
    void run() override { /* 耗时操作 */ }
};
```

**项目中的多线程需求：**

**已经有多线程的地方：**
- `TcpClient.cpp`中使用`CreateThread()`创建接收线程
- 这是必要的，因为接收数据需要阻塞等待

**可以改进的地方：**
- 网络通信可以使用`QTcpSocket`的异步信号-槽，避免手动管理线程
- 耗时操作（如AI计算）可以放到子线程，避免界面卡顿

**建议改进：**
```cpp
// 使用QTcpSocket的异步机制
QTcpSocket* socket = new QTcpSocket(this);
connect(socket, &QTcpSocket::readyRead, 
        this, &CKernel::onReadyRead);

```"

## 四、网络编程问题

### Q4.1：项目中使用的TCP和UDP有什么区别？为什么选择TCP？

**回答思路：** 对比TCP和UDP + 说明项目选择

**参考回答：**

"**TCP和UDP的区别：**

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接性 | 面向连接（需要建立连接） | 无连接 |
| 可靠性 | 可靠（重传、排序、流控） | 不可靠（可能丢包、乱序） |
| 有序性 | 保证数据有序 | 不保证有序 |
| 速度 | 较慢（有握手、确认等开销） | 较快 |
| 应用场景 | 文件传输、网页、邮件 | 视频、游戏、DNS |

**项目中为什么选择TCP？**
1. **可靠性要求高**：游戏操作（准备、落子、胜负判定）不能丢失或乱序
2. **数据量小**：每步棋的信息很小，TCP的开销可以接受
3. **实现简单**：TCP保证可靠传输，不需要自己在应用层实现重传

**UDP在项目中没有被使用**（虽然代码中有`UdpNet`类，但没有实际使用）

**改进建议：**
- 对于实时性要求高的场景（如棋步同步），可以考虑TCP + UDP混合
- 或者使用WebSocket（基于TCP，但更现代）"

**优化方案1：添加长度字段和校验和**

```cpp
// 新协议头部
typedef struct STRU_PACK_HEADER
{
    int type;       // 协议类型
    int length;     // 数据长度（不包括头部）
    int checksum;   // 校验和
}STRU_PACK_HEADER;

// 发送时
void sendPacket(void* data, int dataLen)
{
    // 1. 计算校验和
    int checksum = calculateChecksum(data, dataLen);
    
    // 2. 构造头部
    STRU_PACK_HEADER header;
    header.type = type;
    header.length = dataLen;
    header.checksum = checksum;
    
    // 3. 发送头部+数据
    send(sock, (char*)&header, sizeof(header), 0);
    send(sock, (char*)data, dataLen, 0);
}

// 接收时
void recvPacket()
{
    // 1. 先读头部
    STRU_PACK_HEADER header;
    recv(sock, (char*)&header, sizeof(header), MSG_WAITALL);
    
    // 2. 根据length读数据
    char* data = new char[header.length];
    recv(sock, data, header.length, MSG_WAITALL);
    
    // 3. 验证校验和
    if(calculateChecksum(data, header.length) != header.checksum)
    {
        // 数据损坏！
        delete[] data;
        return;
    }
    
    // 4. 处理数据
    DealData(header.type, data, header.length);
}

## 附录：面试技巧

### 技巧1：STAR法则

在回答行为面试问题时，使用STAR法则：
- **Situation**（情境）：描述遇到的问题或挑战
- **Task**（任务）：说明你需要完成什么任务
- **Action**（行动）：详细描述你采取了哪些行动
- **Result**（结果）：说明最终的结果和收获

---

### 技巧2：展示思考过程

在回答技术问题时，不要只给出答案，要展示你的思考过程：
- "这个问题我有几个方面需要考虑..."
- "首先，我会分析...然后，我会..."
- "如果是我，我会这样做..."

---

### 技巧3：承认不足，但展示学习能力

如果遇到不会的问题，不要慌张：
- "这个问题我没有深入研究过，但我的理解是..."
- "如果让我解决这个问题，我会尝试..."
- "回去后我会深入研究一下这个问题"

---

### 技巧4：准备问题问面试官

面试结束前，面试官通常会问："你有什么问题想问我们的吗？"
- ✅ 问团队技术栈、项目情况
- ✅ 问公司的技术发展计划
- ✅ 问这个职位的成长路径
- ❌ 不要问薪资、福利（等技术面过了再问HR）

---

### 技巧5：保持自信，但不要傲慢

- ✅ 对自己的项目要熟悉，能够清晰地介绍
- ✅ 遇到不会的问题，诚实地承认，但展示学习能力
