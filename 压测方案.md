## 二、压测工具

### 推荐方案：Python 脚本（轻量、易改）

```python
# stress_client.py — 压测客户端模板
import socket
import struct
import time
import threading
import random

SERVER_IP = "127.0.0.1"
SERVER_PORT = 8000

class GomokuClient:
    def __init__(self, client_id):
        self.id = client_id
        self.sock = None

    def connect(self):
        self.sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        self.sock.connect((SERVER_IP, SERVER_PORT))

    def send_pack(self, data):
        """协议格式: 4字节长度 + 数据体"""
        self.sock.send(struct.pack('<i', len(data)) + data)

    def register(self, tel, name, password):
        """注册请求 (type=10000)"""
        pack = struct.pack('<i40s40s40s', 10000,
                          tel.encode().ljust(40, b'\0'),
                          name.encode().ljust(40, b'\0'),
                          password.encode().ljust(40, b'\0'))
        self.send_pack(pack)

    def login(self, tel, password):
        """登录请求 (type=10002)"""
        pack = struct.pack('<i40s40s', 10002,
                          tel.encode().ljust(40, b'\0'),
                          password.encode().ljust(40, b'\0'))
        self.send_pack(pack)

    def heartbeat(self, userid, zoneid, roomid):
        """心跳包 (type=10025)"""
        pack = struct.pack('<iiii', 10025, userid, zoneid, roomid)
        self.send_pack(pack)

    def join_zone(self, userid, zoneid):
        """加入专区 (type=10004)"""
        pack = struct.pack('<iii', 10004, userid, zoneid)
        self.send_pack(pack)

    def join_room(self, userid, roomid):
        """加入房间 (type=10006)"""
        pack = struct.pack('<iii', 10006, userid, roomid)
        self.send_pack(pack)

    def recv_loop(self):
        """接收线程"""
        while True:
            try:
                len_data = self.sock.recv(4)
                if not len_data: break
                pack_len = struct.unpack('<i', len_data)[0]
                data = b''
                while len(data) < pack_len:
                    data += self.sock.recv(pack_len - len(data))
                # 处理数据...
            except:
                break
```

### 备选方案

| 工具 | 适用 | 缺点 |
|------|------|------|
| Python + raw socket | 自定义二进制协议 | 需要自己写协议编解码 |
| `wrk` / `ab` | HTTP 服务压测 | **不适用**（非 HTTP 协议） |
| `tc` (traffic control) | 模拟网络延迟/丢包 | 配合其他工具使用 |
| `sysbench` | MySQL 压力测试 | 单独测数据库 |

## 三、压测场景

### 场景 1：连接风暴

```
目标: 验证 accept 和 epoll 处理能力
操作: 500个客户端同时 connect → 登录 → 保持心跳
监控: Server 内存、CPU、epoll_wait 返回的 nfd
预期: 500 连接建立成功，CPU < 50%，epoll 无 error
```

**监控命令**：
```bash
# 服务端进程
top -p $(pidof PlayHallServer)

# 文件描述符数
ls /proc/$(pidof PlayHallServer)/fd | wc -l

# epoll 态（没有内置计数，用 strace 近似看）
strace -e epoll_wait -p $(pidof PlayHallServer) 2>&1 | head -20
```

### 场景 2：心跳风暴

```
目标: 验证心跳处理不丢包、不误判超时
操作: 500 客户端持续发心跳 30 分钟
     随机挑选 20 个客户端停止心跳 → 15s 后应被 HandleDisconnect 清理
监控: 心跳超时误判率、HandleDisconnect 调用次数
预期: 20 个掉线客户端被准确标记（无漏判、无误判）
```

### 场景 3：高并发落子

```
目标: 测试落子→转发→渲染全链路延迟
操作: 121 个房间全部满员（2人+旁观）
     每个房间持续对局（按真实落子节奏模拟，非洪水）
监控: 服务端 SendData 耗时、客户端收到延迟
预期: 每步落子转发延迟 <100ms
```

### 场景 4：极限旁观

```
目标: 观众数量对落子延迟的影响
操作: 1 个房间，2 人对局，998 人旁观（共 1000 连接）
     发 50 步落子，记录最后 1 个旁观者收到的时间
监控: SendData 的序列化耗时
预期: 1000 人广播 <50ms（非阻塞 send，瓶颈不在遍历）
```

### 场景 5：内存泄漏检测

```
目标: 长时间运行无泄漏
操作: 100 客户端反复登录→进入房间→退出→掉线 → 循环 2 小时
监控: valgrind 或 RSS 趋势图
预期: 内存稳定，无持续增长
```

### 场景 6：数据库压力

```
目标: 并发登录+注册的 MySQL 处理能力
操作: 100 线程并发注册（随机手机号）+ 登录
监控: MySQL 慢查询日志、服务端 LoginRq 耗时

```

### 场景 7：网络异常模拟

```
目标: 验证心跳超时和重连恢复
操作: 用 tc 注入网络异常

# 丢包 30%

监控: 重连成功率、token 校验、棋局恢复完整性
预期: 重连后棋局面貌一致，不掉步
```

## 四、服务端瓶颈分析

| 瓶颈点 | 当前实现 | 风险 |
|--------|---------|------|
| epoll 事件处理 | epoll + EPOLLONESHOT + 线程池 | ✅ 良好 |
| 连接数上限 | `MAX_EVENTS=4096` | 单机可 4000+ 连接 |
| 房间数 | 121 固定 `vector<list<int>>(121)` | 够用 |
| 落子广播 | 链表串行 `send()` | ≤1000 人非阻塞下无压力 |
| 数据库 | 单连接 `CMysql`，无连接池 | ⚠️ 高并发登录/注册可能排队 |
| 心跳扫描 | 独立线程 10s 一轮 | 遍历 `m_heartbeatMap`，O(n) 小 |
| 内存 | `new USER_INFO`，心跳超时 `delete` | ⚠️ 自然断线无回调（已修复） |

## 五、快速验证脚本

```python
# quick_test.py — 基础连接+心跳验证
import socket, struct, time, threading, sys

N = 100  # 并发数
IP, PORT = "127.0.0.1", 8000

results = {"ok": 0, "fail": 0}

def one_client(i):
    try:
        s = socket.socket(); s.settimeout(10)
        s.connect((IP, PORT))
        # 发心跳 10 轮
        hb = struct.pack('<iiii', 10025, i+1, 0, 0)
        for _ in range(10):
            s.send(struct.pack('<i', 16) + hb)
            time.sleep(5)
        s.close()
        results["ok"] += 1
    except Exception as e:
        results["fail"] += 1
        print(f"[{i}] {e}")

threads = [threading.Thread(target=one_client, args=(i,)) for i in range(N)]
[t.start() for t in threads]
[t.join() for t in threads]
print(f"ok={results['ok']} fail={results['fail']}")
```

**运行**：
```bash
python3 quick_test.py
# 预期: ok=100 fail=0
```

## 七、服务端性能检测命令

压测时在服务端机器上开多个终端，各跑一个。

### 1. 整体 CPU / 内存

```bash
# 实时 top
top -p $(pidof PlayHallServer)

# 每秒一行，写日志
vmstat 1 | tee perf.log
```

### 2. 内存 RSS 趋势（检测泄漏）

```bash
while true; do
  awk '/VmRSS|VmSize/ {printf "%s %s\n", strftime("%H:%M:%S"), $0}' \
    /proc/$(pidof PlayHallServer)/status
  sleep 5
done | tee mem.log
```

### 3. 文件描述符数（是否泄漏）

```bash
watch -n 2 "ls /proc/\$(pidof PlayHallServer)/fd | wc -l"
# 压测结束后应回落，不回落 = fd 泄漏
```

### 4. 线程数

```bash
watch -n 2 "ls /proc/\$(pidof PlayHallServer)/task | wc -l"
```

### 5. epoll 是否有 error

```bash
# strace 抓 epoll_wait 返回值（-1 = error, ENOMEM/EINTR 等）
strace -e trace=epoll_wait -p $(pidof PlayHallServer) 2>&1 | tee epoll.log
# 压测完后 grep "= -1" epoll.log，应为空
```

### 6. 网络连接数

```bash
# ESTABLISHED 连接数
watch -n 2 "ss -tn state established '( dport = :8000 or sport = :8000 )' | wc -l"

# 各状态统计
ss -s
```

### 7. 系统调用耗时分布

```bash
# 哪类系统调用最耗时（accept / recv / send / epoll_wait）
strace -c -p $(pidof PlayHallServer)
# Ctrl+C 停止后打印汇总表
```

### 8. 一键启动全部监控

```bash
#!/bin/bash
# monitor.sh — 压测前置脚本
PID=$(pidof PlayHallServer)
if [ -z "$PID" ]; then echo "Server not running"; exit 1; fi

# 终端 1: top
xterm -e "top -p $PID" &

# 终端 2: 内存日志
xterm -e "while true; do awk '/VmRSS/{printf \"%s %s\n\", strftime(\"%H:%M:%S\"), \$0}' /proc/$PID/status; sleep 5; done | tee mem.log" &

# 终端 3: fd 数
xterm -e "watch -n 2 'ls /proc/$PID/fd | wc -l'" &

# 终端 4: 连接数
xterm -e "watch -n 2 'ss -tn state established dport = :8000 | wc -l'" &

echo "PID=$PID 监控已启动"
```

## 八、压测完怎么判断

| 检查项 | 正常 | 异常信号 |
|--------|------|---------|
| `epoll_wait` 无 error | `epoll.log` 中无 `= -1` | 出现 ENOMEM/EINTR/EBADF |
| fd 数不涨 | 压测结束回落到基线 | 持续增长不回退 |
| 内存不涨 | RSS 在 ±10MB 波动 | 持续上升不回落 |
| CPU 不爆 | <80% | 100% 卡死 |
| 心跳不误判 | timeoutUsers=20（故意掉的） | 多杀了正常连接 |
| `dmesg` 无 OOM | `dmesg \| tail` 无 oom-killer | 进程被杀 |
